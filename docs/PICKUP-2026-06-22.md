## 2. Release / CI / repo hygiene (small, high-value)

### 2.1 `publish.yml` tag-CI red on manual releases — **DONE (dev-beta `5c7e0cc`)**
The `v*` tag re-ran `vsce publish` on an already-(manually-)published version →
failed every release (v3.6.2/3/4 red). Fixed: publish + GitHub-release steps are
now gated behind the repo Actions var `CI_PUBLISH` (default off), so a tag builds
+ tests + packages + uploads the `.vsix` artifact GREEN with publishing staying
manual; package step uses `--no-dependencies`. Takes effect at the next release
(when the fix reaches a tagged master commit). To let CI publish a future tag:
set repo var `CI_PUBLISH=true` and don't pre-publish that version.

### 2.2 Branch + worktree pruning (deferred by policy) — **[verified 06-22]**
